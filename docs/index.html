Agent Flow

CrewAI 에이전트 플로우

5개의 AI 에이전트가 3단계로 협업하여 면접을 진행합니다.

Phase 1 — 면접 준비 (1회)
📋
Analyst
이력서/JD 분석가
입력: 이력서 + JD + MD지식
출력: 후보자 프로필 + 기술 매트릭스
→
🎯
Planner
질문 설계자
입력: 후보자 프로필
출력: 질문 시나리오 (8~10문항, JSON)
→
Phase 2 — 면접 진행 (매 턴)
🎙️
Interviewer
기술 면접관
입력: 시나리오 + 사용자 답변
출력: 다음 질문 or 꼬리질문
⇄
⚖️
Evaluator
답변 평가자
입력: 답변 + MD지식 + 채점기준
출력: 점수(1~10) + 피드백
→
Phase 3 — 리포트 (1회)
📊
Coach
종합 코치
입력: 전체 면접 로그 + 평가 데이터
출력: 종합 리포트 + 학습 추천
Analyst → Planner는 면접 시작 전 1회 실행 (@Async 백그라운드). Interviewer ⇄ Evaluator는 매 턴마다 반복 실행. Coach는 면접 종료 후 1회 실행.
Session Flow

면접 세션 흐름

3단계 파이프라인으로 면접이 진행됩니다.

Phase 1 — 면접 준비
  • 이력서(PDF) + JD 업로드
  • POST /api/interview/create
  • @Async 백그라운드 준비 시작
  • Analyst (병렬 3태스크) → Planner
  • 질문 시나리오 JSON 생성
  • 프론트: 5초 간격 폴링으로 상태 확인
→
Phase 2 — 면접 진행 (N회)
  • 사용자 "입장하기" 클릭
  • 메인 질문 8~10개 순차 진행
  • POST /api/interview/{id}/answer
  • Evaluator → 채점 (1~10점)
  • Interviewer → 꼬리질문 or 다음 질문
  • 메인 질문당 꼬리질문 최대 2회
→
Phase 3 — 리포트 생성
  • POST /api/interview/{id}/end
  • Coach 에이전트 실행
  • 종합 점수 + 등급 (A/B/C/D/F)
  • 강점/보완점 정리
  • 지식 갭 분석 (학습O/X 분류)
  • 학습 추천 + 다음 단계 제시
상태: PREPARING → READY → IN_PROGRESS → COMPLETED | PREPARING → FAILED → 재시도
Features

핵심 기능 상세

프로젝트의 기술적 차별점을 소개합니다.

01

RAG 기반 문맥 질문 생성

pgvector의 similarity_search로 사용자의 이력서, JD, MD 스터디 노트를 교차 검색하여 개인화된 면접 질문을 생성합니다.

기술: pgvector + OpenAI Embedding + LangChain RAG Pipeline
02

비동기 면접 준비

AI 질문 시나리오 생성(10~30초)을 @Async 백그라운드로 처리하고, 프론트엔드에서 5초 간격 폴링으로 상태를 확인합니다.

패턴: @Async + Frontend Polling + Status State Machine
03

지식 갭 분석

사용자의 MD 노트와 면접 답변을 대조하여 "학습O+답변O", "학습O+답변X", "미학습" 3단계로 지식 갭을 분류합니다.

분류: studied_and_strong / studied_but_weak / not_studied
API Design

API 설계

Spring Boot와 FastAPI 간 REST API 구성

Authentication
POST /api/auth/signup
회원가입 (이메일/비밀번호)
POST /api/auth/login
로그인 → JWT Access Token 발급
Interview (모의면접)
POST /api/interview/create
면접 생성 (Multipart: PDF + JD) → 즉시 응답 + @Async 준비
GET /api/interview
면접 세션 목록 (폴링용, status 포함)
POST /api/interview/{id}/enter
면접 입장 (READY → IN_PROGRESS)
POST /api/interview/{id}/answer
답변 제출 → 평가 + 다음 질문 반환
POST /api/interview/{id}/end
면접 종료 → Coach 리포트 생성
GET /api/interview/{id}/report
면접 리포트 조회
Knowledge (지식 관리)
POST /api/knowledge/upload
MD 파일 수신 → 태그 파싱 + 임베딩
GET /api/knowledge
업로드된 지식 노트 목록 조회
Quiz (문제 생성)
POST /api/quiz/generate
태그/난이도 기반 면접형 문제 생성
POST /api/quiz/{id}/submit
답변 제출 → 채점 + 누락 포인트 + 관련 노트
Internal API (Spring Boot ↔ FastAPI)
POST /ai/interview/prepare
이력서+JD+RAG → 질문 시나리오 생성 (Phase 1)
POST /ai/interview/evaluate
답변 평가 + 다음 질문 결정 (Phase 2)
POST /ai/interview/report
전체 로그 → 종합 리포트 생성 (Phase 3)
POST /ai/knowledge/embed
MD 텍스트 → 벡터 임베딩 저장
Database

데이터베이스 스키마

PostgreSQL 16 + pgvector — 7개 테이블

users
  • idUUID PK
  • emailVARCHAR
  • passwordVARCHAR
  • nameVARCHAR
  • created_atTIMESTAMP
resumes
  • idUUID PK
  • user_idUUID FK
  • file_pathVARCHAR
  • parsed_textTEXT
  • created_atTIMESTAMP
FK: user_id → users.id
knowledge_notes
  • idUUID PK
  • user_idUUID FK
  • file_nameVARCHAR
  • contentTEXT
  • tagsTEXT[]
  • embeddingVECTOR(1536)
FK: user_id → users.id
interview_sessions
  • idUUID PK
  • user_idUUID FK
  • resume_idUUID FK
  • jd_textTEXT
  • difficultyVARCHAR
  • statusENUM
  • question_scenarioJSONB
FK: user_id → users.id, resume_id → resumes.id
interview_turns
  • idUUID PK
  • session_idUUID FK
  • turn_numberINT
  • question_textTEXT
  • question_typeENUM
  • user_answerTEXT
  • scoreINT
  • feedbackTEXT
FK: session_id → interview_sessions.id
interview_reports
  • idUUID PK
  • session_idUUID FK
  • overall_scoreINT
  • strengthsJSONB
  • improvementsJSONB
  • study_recommendationsJSONB
FK: session_id → interview_sessions.id (1:1)
quiz_attempts
  • idUUID PK
  • user_idUUID FK
  • knowledge_note_idUUID FK
  • question_textTEXT
  • user_answerTEXT
  • is_correctBOOLEAN
  • scoreINT
  • missing_pointsJSONB
FK: user_id → users.id, knowledge_note_id → knowledge_notes.id
